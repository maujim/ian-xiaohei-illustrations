# QA Checklist

## Must-Pass Items

- It is 16:9 horizontal.
- The background is clean white.
- Xiaohei is present.
- Xiaohei carries the core action and is not just decoration.
- The composition does not copy an old example; it creates a new metaphor for the current article.
- The image is absurd, creative, and interesting.
- It is simple and clean; the main subject does not exceed about 60% of the canvas.
- One image explains only one core structure.
- English labels are few, short, and readable.
- Orange is used only for the main path or arrows.
- Red is used only for emphasis, problems, reminders, or results.
- Blue is used only for supplemental explanation, feedback, or system state.

## Failure Signals

Regenerate or locally edit if any of the following appear:

- The top-left corner has titles such as "Common Pitfalls / Workflow / System Architecture Diagram / Roadmap".
- Xiaohei looks like a mascot, meme character, or cute cartoon.
- The image looks like a PPT, course slide, or formal flowchart.
- Too many elements, arrows, or nodes.
- Text turns into long explanations.
- The background has paper texture, shadows, gradients, beige, or noise.
- Real UI screenshots or tech-style interfaces.
- English spelling errors are severe or labels are unreadable.
- The image is too rigid and has no absurd metaphor.
- The composition is too similar to old examples in `assets/examples/`.

## Iteration Method

- Too ordinary: make Xiaohei the action subject and add a strange but valid metaphor.
- Too complex: remove nodes and keep only one action plus 3-5 short labels.
- Too cute: emphasize deadpan, blank serious expression, not cute, not mascot.
- Too PPT-like: remove titles, borders, tidy grids, and excessive arrows; turn it into a hand-drawn scene.
- Too similar to an old example: keep the core meaning, but change the main object and Xiaohei's action.
- Text errors: prioritize local editing; if there are many errors, regenerate and reduce the number of labels.

## Delivery Evaluation

A high-quality image should first make readers think "a little strange", then let them understand the structure within one second.

If the first glance feels like a tutorial page instead of an absurd product sketch on white paper, it fails.
